# Skill Best Practices

This skill provides a canonical template and best‑practice checklist for writing SKILL.md files used by Raven.

## Overview

- **Purpose** – Define a clear, consistent layout so new skills are easy to understand, maintain, and share.
- **Audience** – Skill authors and maintainers.

## Recommended Layout

1. **Front‑matter** – YAML block at the top with `name`, `description`, and optional `metadata` (emoji, requirements, install steps).
2. **Title** – `# Skill Name` matching the `name` field.
3. **Brief Description** – One‑paragraph summary.
4. **Sections** – Use clear headings:
   - `## Overview`
   - `## Usage`
   - `## References`
   - `## Guardrails`
5. **Code Samples** – fenced code blocks with language tags.
6. **Links** – Use relative paths for internal docs; absolute URLs for external resources.

## Front‑matter Details

```yaml
---
name: <skill‑name>
description: <short description>
metadata:
---
```

- Keep keys alphabetical.
- Only include fields that are needed.

## Content Sections

### Overview
Explain what the skill does and when to use it.

### Usage
Provide step‑by‑step instructions or a quick‑start guide.

### References
List any supporting documents, scripts, or external links.

### Guardrails
Document safety considerations, required permissions, and any “do not” statements.

## Example Skeleton

```markdown
---
name: my‑skill
description: Short description of what the skill does.
metadata:
---

# My Skill

## Overview
...
## Usage
...
## References
...
## Guardrails
...
```

## Validation Checklist

- [ ] Front‑matter present and valid YAML.
- [ ] `name` matches the skill directory name.
- [ ] Description is concise (<120 characters).
- [ ] All required binaries listed under `requires.bins`.
- [ ] Installation steps are accurate and testable.
- [ ] Sections follow the recommended order.
- [ ] Guardrails clearly state any safety constraints.
